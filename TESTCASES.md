# Тест-кейсы

## API эндпоинты

| Метод | Endpoint | Что делает |
|-------|----------|------------|
| POST | `/api/1/item` | Создать объявление |
| GET | `/api/1/item/:id` | Получить по ID |
| GET | `/api/1/:sellerID/item` | Все объявления продавца |
| GET | `/api/1/statistic/:id` | Статистика объявления |
| DELETE | `/api/2/item/:id` | Удалить объявление |
| GET | `/api/2/statistic/:id` | Статистика v2 |

Тело POST-запроса:
```json
{
  "sellerID": 123456,
  "name": "string",
  "price": 1000,
  "statistics": { "likes": 0, "viewCount": 0, "contacts": 0 }
}
```

## API тесты (Задание 2.1)

### Создание объявления (POST /api/1/item)

**Позитивные:**

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-001 | Валидные данные — всё заполнено корректно | 200, id возвращён, поля сохранены |
| TC-002 | Цена = 0 (граница) | 200 или 400 с ошибкой |
| TC-003 | Цена = 999999999 (макс) | 200 |
| TC-004 | Статистика нулями | 200 |
| TC-005 | Статистика ненулевая (likes:10, viewCount:50, contacts:5) | 200, данные сохранены |
| TC-006 | Несколько объявлений одного продавца → GET список | Все возвращаются |
| TC-007 | Разные sellerID → разные id | Уникальные id |
| TC-008 | Кириллица в имени («Персидский котёнок») | 200, имя сохранилось |

**Негативные:**

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-010 | Нет sellerID | 400 |
| TC-011 | Нет name | 400 |
| TC-012 | Нет price | 400 |
| TC-013 | Пустое тело `{}` | 400 |
| TC-014 | Цена отрицательная (-100) | 400 |
| TC-015 | Статистика отрицательная (likes:-5) | 400 |
| TC-016 | sellerID = 0 | 400 |
| TC-017 | sellerID строкой `"111111"` | 400 или конвертация |
| TC-018 | price строкой `"1000"` | 400 или конвертация |
| TC-019 | Content-Type: text/plain, не JSON | 400 |
| TC-020 | name = пустая строка | 400 |
| TC-021 | SQL-инъекция в name | Не падает |
| TC-022 | XSS в name | Экранируется |
| TC-050 | POST на GET-endpoint | 405 |

**Корнер-кейсы:**

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-025 | Два одинаковых объявления | Два разных id |
| TC-026 | 100 объявлений подряд | Все созданы |
| TC-027 | price = null | 400 |
| TC-030 | Эмодзи в имени | 200 |
| TC-031 | Иероглифы в имени | 200 |

### Получение объявления (GET /api/1/item/:id)

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-040 | Существующее объявление | 200, данные совпадают |
| TC-041 | Типы данных в ответе | id — строка, price — число и т.д. |
| TC-043 | Несуществующий id | 404 |
| TC-044 | Пустой id | 400 |
| TC-045 | SQL-инъекция в id | Не падает |

### Объявления продавца (GET /api/1/:sellerID/item)

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-060 | 3 объявления одного продавца → список | Массив из 3 |
| TC-061 | 1 объявление → список | Массив из 1 |
| TC-062 | Продавец без объявлений | Пустой массив или 404 |
| TC-063 | sellerID = "abc" | 400 |
| TC-064 | sellerID = -1 | 400 |

### Статистика (GET /api/1/statistic/:id, /api/2/statistic/:id)

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-070 | Статистика существующего объявления | 200 |
| TC-071 | Статистика совпадает с отправленной при создании | Данные совпали |
| TC-072 | v2 API | 200, совместимый формат |
| TC-073 | Несуществующий id | 404 |

---

### Удаление (DELETE /api/2/item/:id)

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-080 | Удаление существующего → проверка GET | DELETE 200, потом GET 404 |
| TC-081 | Повторное удаление | 404 или 200 (идемпотентный) |
| TC-082 | Удаление несуществующего | 404 |

---

### Нефункциональные

| ID | Что проверяю | Ожидание |
|----|--------------|----------|
| TC-090 | Время ответа всех endpoints | < 5 сек |
| TC-091 | CORS (Origin: http://evil.com) | Не пропускает чужие Origin |
| TC-092 | Без Content-Type | 400 или 415 |
| TC-093 | Content-Type: application/x-www-form-urlencoded | 400 или 415 |

### E2E сценарии

| ID | Сценарий |
|----|----------|
| TC-100 | Полный цикл: создать → получить → статистика → удалить → убедиться что удалено |
| TC-101 | Создать 5 для одного продавца → получить список → проверить каждый |
| TC-102 | Создать 3 → удалить одно → проверить что в списке осталось 2 |
