# Тест-кейсы

## API эндпоинты

POST /api/1/item — создать объявление
GET /api/1/item/:id — получить по ID
GET /api/1/:sellerID/item — все объявления продавца
GET /api/1/statistic/:id — статистика v1
DELETE /api/2/item/:id — удалить
GET /api/2/statistic/:id — статистика v2

Тело запроса:
```json
{
  "sellerID": 123456,
  "name": "string",
  "price": 1000,
  "statistics": { "likes": 0, "viewCount": 0, "contacts": 0 }
}
```

## API тесты

### Создание объявления

Позитивные:

- TC-001 — валидные данные — 200, id возвращён
- TC-002 — цена 0 — 200 или 400 (техника: граничные значения)
- TC-003 — цена 999999999 — 200 (граничные значения)
- TC-004 — статистика нулями — 200 (эквивалентное разделение)
- TC-005 — статистика ненулевая — данные сохранены (эквивалентное разделение)
- TC-006 — несколько объявлений одного продавца (комбинаторика)
- TC-007 — разные sellerID, уникальные id (классы эквивалентности)
- TC-008 — кириллица в имени (именное разделение)

Негативные:

- TC-010 — нет sellerID — 400
- TC-011 — нет name — 400
- TC-012 — нет price — 400
- TC-013 — пустое тело {} — 400
- TC-014 — цена -100 — 400 (негативные значения)
- TC-015 — статистика отрицательная — 400
- TC-016 — sellerID = 0 — 400 (граничные значения)
- TC-017 — sellerID строкой — 400 или конвертация (несоответствие типов)
- TC-018 — price строкой — 400 или конвертация
- TC-019 — Content-Type: text/plain — 400
- TC-020 — name пустая строка — 400 (граничные значения)
- TC-021 — SQL-инъекция — не падает (инъекции)
- TC-022 — XSS — экранируется (инъекции)
- TC-050 — POST на GET-endpoint — 405 (неправильный метод)

Корнер-кейсы:

- TC-025 — два одинаковых объявления — два разных id (идемпотентность)
- TC-026 — 100 объявлений подряд — все созданы (нагрузка)
- TC-027 — price = null — 400 (граничные значения)
- TC-030 — эмодзи в имени — 200 (unicode)
- TC-031 — иероглифы в имени — 200 (unicode)

### Получение объявления

- TC-040 — существующее объявление — 200 (E2E)
- TC-041 — типы данных в ответе (E2E)
- TC-043 — несуществующий id — 404 (классы эквивалентности)
- TC-044 — пустой id — 400 (граничные значения)
- TC-045 — SQL-инъекция в id — не падает (инъекции)

### Объявления продавца

- TC-060 — 3 объявления одного продавца — массив из 3 (E2E)
- TC-061 — 1 объявление — массив из 1 (E2E)
- TC-062 — продавец без объявлений — пустой массив или 404 (классы эквивалентности)
- TC-063 — sellerID = "abc" — 400 (несоответствие типов)
- TC-064 — sellerID = -1 — 400 (граничные значения)

### Статистика

- TC-070 — существующее объявление — 200 (E2E)
- TC-071 — статистика совпадает с отправленной (E2E)
- TC-072 — v2 API — совместимый формат (E2E)
- TC-073 — несуществующий id — 404 (классы эквивалентности)

### Удаление

- TC-080 — удаление существующего — DELETE 200, GET 404 (E2E)
- TC-081 — повторное удаление — 404 или 200 (идемпотентность)

### Нефункциональные

- TC-090 — время ответа < 5 сек
- TC-091 — CORS (Origin: http://evil.com) — не пропускает
- TC-092 — без Content-Type — 400 или 415
- TC-093 — form-urlencoded — 400 или 415

### E2E сценарии

- TC-100 — полный цикл: создать, получить, статистика, удалить, проверить
- TC-101 — создать 5 для одного продавца, проверить список и каждый по ID
- TC-102 — создать 3, удалить одно, осталось 2

## UI тесты

### Главная страница

- UI-001 — фильтр по цене [1000, 50000] (граничные значения)
- UI-002 — мин > макс — ошибка (обратная совместимость)
- UI-003 — сортировка по цене, возрастание (комбинаторика)
- UI-004 — сортировка по цене, убывание (комбинаторика)
- UI-005 — фильтр по категории (эквивалентное разделение)
- UI-006 — тогл «Только срочные» вкл (переключение состояния)
- UI-007 — тогл «Только срочные» выкл (переключение состояния)

### Страница статистики

- UI-010 — кнопка «Обновить» — статистика обновилась (E2E)
- UI-011 — кнопка «Остановить» — автообновление прекратилось
- UI-012 — кнопка «Запустить» — автообновление пошло снова

### Мобильная версия

- UI-020 — переключение темы на 375x667 (адаптивное тестирование)
- UI-021 — тема сохраняется после перезагрузки (адаптивное тестирование)

### Карточка объявления

- UI-030 — клик на объявление — открылась /item/:id (E2E)
- UI-031 — данные в карточке (название, цена, статистика)
